Scan ascii stl file and perform check on format error.

Parameters
[in]file_handleinput stream from stl file.
[in,out]err_mapfor each solid found in the stl file, store a bitmask for format errors encountered in the solid data section. error code: 0 -> unterminated solid block 1 -> undeterminated facet block 2 -> normal data are missing 3 -> wrong number of components for facet normal 4 -> wrong number of vertices in facet block 5 -> wrong number of coordinates for vertex
Returns
returns and error flag for I/O errors: err = 0: no error(s) encountered err = 1: failed to scan/check stl file

)

Read data from ascii stl file.

Parameters
[in,out]file_handlestream from stl file
[in,out]nVon input stores the number of vertices currently stored in V. On output stores the input value increased by the number of vertices acquired from the stl file.
[in,out]nTon input stores the number of facets->vertex connectivity entries currently stored in T. On output stores the input value increased by the number of facet acquired from the stl file.
[in,out]Vvertex coordinate list. On output stores the coordinates of vertices acquired from the stl file. Vertices acquired from the stl file are appended at the end of V (no distinction is made if multiple solids are stored in the same stl file).
[in,out]Nfacet's normals. On output stores the normal unit vector to each facet acquired from the stl file. Normals acquired from the stl file are appended at the end of N (no distinction is made if multiple solids are stored in the same stl file).
[in,out]Tfacet->vertex connectivity. On output stores facet->vertex connectivity for each facet acquired from the solid. New connectivity entries are appended at the end of T (no distinction is made if multiple solids are stored in the same stl file).
Returns
returns an error flag for I/O errors: err = 0: no error(s) encountered err = 1: failed to read from input stream

Read STL facet data from ascii stl file.

Parameters
[in,out]file_handleinput stream from stl file
[in,out]nVon input stores the number of vertices previously acquired from the stl file, on output stores the input value incremented by the number of vertices read from the facet section.
[in,out]nTon input stores the number of facets previously aquired from the stl file, on output stores the input value incremented by 1.
[in,out]Vvertex coordinate list. Vertices read from facet section are stored in the location V[nV], V[nV+1], etc. nV is the number of vertices previously acquired from the stl file and passed as input to this function.
[in,out]Nnormal list. Normal read from facet section is stored at position N[nT], where nT is the number of facet previously acquired from the stl file and passed as input to this function.
[in,out]Tvertex->facet connectivity. A new connectivity entry for the facet begin read is created at position T[nT], where nT is the number of facet previously acquired from the stl file and passed as input to this function.
Returns
returns an error flag for I/O errors: err = 0: no error(s) encountered err = 1: failed to read from input stream err = 2: one facet has more than 3 vertices
